CricketQL

Ask questions in English and get verified answers from a real database — not an LLM guess. A production-grade NLP → SQL system with strict safety guarantees and real PostgreSQL execution.

Try Live Demo View GitHub

Try: “Who has the highest batting average in T20 matches?”


Responses not sourced from the database are explicitly marked.

CricketQL is a domain-specific implementation of a general, safe NLP → SQL architecture that can be adapted to any structured dataset. It converts natural language into safe, validated SQL queries executed on a real PostgreSQL database.

If data is missing, an optional external LLM can answer — clearly labeled and fully controlled.

How it works

1. Natural Language Input

User asks a cricket-related question in plain English.

2. Guarded LLM Parsing

LLM is restricted to cricket-only, SELECT-only SQL generation.

3. PostgreSQL Execution

Validated SQL is executed on real cricket datasets.

4. Optional LLM Fallback

If data is missing, an external LLM answers transparently.

Key features

Domain-restricted NLP → SQL

Prevents off-topic queries and hallucinated schemas.

Query safety

Blocks INSERT, UPDATE, DELETE at both LLM and backend level.

Real database execution

No mock data. Real PostgreSQL queries.

Modular Django backend

Clean, scalable architecture.

Under the Hood

  • Relational PostgreSQL schema with batting, bowling, and fielding tables
  • Natural language → SQL generation with backend-level validation
  • Strict SELECT-only query enforcement (no mutations)
  • Rejection of ambiguous, unsafe, or out-of-domain prompts
  • Optional LLM fallback clearly labeled as non-database output
